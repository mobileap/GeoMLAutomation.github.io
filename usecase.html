Flood Extent Mapping

During the 2023 monsoon season, rapid and accurate delineation of flood-affected zones was critical for emergency response teams. Traditional manual digitising was too slow. This research explored whether SAR-based deep learning could produce reliable flood maps within hours of image acquisition.

▶
Demo · 0:42 · End-to-end pipeline walkthrough

// Pipeline

01

Input · Sentinel-1 SAR

C-band SAR GRD product, 10m resolution, VV+VH polarisation. Acquired within 6 hours of flood peak. Downloaded via Copernicus Open Access Hub API.

Sentinel-1C-band10mVV+VH
↓
02

Pre-processing · SNAP + Python

Radiometric calibration, terrain correction (SRTM DEM), speckle filtering using Lee filter, and dB conversion. Automated using SNAP Graph Builder via subprocess in Python.

SNAPGDALLee FilterSRTM
↓
03

Model · U-Net Segmentation

Custom U-Net trained on Sen1Floods11 dataset. Binary segmentation — water vs non-water. Key challenge: handling mixed pixels at flood boundary in agricultural areas.

PyTorchU-NetSen1Floods11Binary segmentation
↓
04

Post-processing · Vectorisation

Raster mask polygonised using GDAL, simplified with Douglas-Peucker, then loaded into PostGIS. Confidence scores attached per polygon based on model output probability.

GDALPostGISDouglas-Peucker
↓
05

Output · Interactive Web Map

Flood polygons served via GeoServer WFS, visualised in Leaflet with time-slider control. Users can toggle satellite basemap, query individual polygons, and export affected area stats.

Leaflet.jsGeoServerWFSPython Flask
